And here we are... at the end of the line (for now). High-level programming languages represent an extreme application of the terms of abstraction and high-level constructs. So what's the difference between this idea and what happens in Java? Java, you see, is object oriented, but also relies on primitives represented directly into memory.

Ruby, by contrast, is entirely object-oriented. It has no primitives and everything can be treated as an object. In essence, the transition from mid-level to high-level abstraction has allowed for more dynamic languages with limitless flexibility in algorithm design.

Key Characteristics:

  • Dynamic Constructs - open classes, message-style methods, etc.
  • Poor performance (seriously!)
  • No frills code, very sparse and to the point.
  • Flexible syntax...
  • Hybrid paradigm! (It can be object-oriented AND functional)
  • Insanely fanatic community keeps it viable.
