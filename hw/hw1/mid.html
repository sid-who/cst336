And in this point in our story, things begin to get interesting. Abstraction, in earnest, comes into play in mid-level languages such as Java and C#. At this level, it becomes necessary to run builds through what is known as a 'virtual machine'. The virtual machine acts as an execution environment, a mediator betweeen the code you've created and the hardware trying to run it. Additionally, mid-level languages are often bootstrapped from low-level languages.

Key Characteristics:

  • A high degree of abstraction for the creation of objects and allow for object-oriented-programming.
  • "Garbage Collection" - a form of automated memory management for ease of use.
  • Most commonplace in workplace environments.
  • Easy to read program flow.
    •

    Garbage collection, in many ways, is why C++ does not qualify as a mid-level language. Garbage Collection must be performed manually in C++, but not C#.
